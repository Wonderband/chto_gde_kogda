Safety & Override Mechanisms — all call triggerReadQuestion()
Mechanism Trigger Condition Implementation Notes
AUTO — Transcript AI says "Внимание! Вопрос!" (both words in same response turn) onTranscriptDelta accumulates → match → triggerPending=true → wait response.done ~300ms advance notice before players hear phrase. Per-turn buffer prevents false match from earlier turns.
MANUAL — Enter key Host presses Enter during SPINNING or READING state triggerReadQRef.current() called directly from keydown handler Immediate — no waiting for response.done. cancelResponse() called defensively.
FALLBACK — 50s timer 50 seconds elapsed since session opened, no transcript trigger fired dialogTimerRef = setTimeout(triggerReadQRef.current, 50_000) Catches cases where AI never says trigger phrase (e.g. different wording, API quirk).
SAFETY — 120s timeout Session-level timeout (RealtimeSession PRE_SESSION_TIMEOUT) _handleError('safety timeout') → onError → send(EVENTS.READING_DONE) Last resort. If READING or SPINNING state, forces game forward. Session always closes cleanly.
⚠ Question text is withheld during Phase 0–1. The preCtx object sent in session instructions includes character, round_type, sector_number but NOT question_text. The question is injected only at Phase 2 Step ⑦ via injectSystemMessage(), ensuring the AI cannot accidentally reveal it during small talk. The QuestionCard also shows animated dots (not text) while game state is READING.
