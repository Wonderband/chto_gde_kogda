Для реалізації вашої гри «Що, Де, Коли» на базі **OpenAI Realtime API** з використанням моделі **GPT-5 Mini**, найкращим підходом буде використання **OpenAI Agents SDK**. Це дозволить вам легко керувати станами діалогу, підключати інструменти (tools) для роботи з JSON та гнучко перемикати режими прослуховування 1-3.  
Ось детальний план впровадження вашої гри:

### 1\. Налаштування середовища та підключення

Перш за все, вам потрібно підготувати технічну базу:

* **Отримання ключа:** Ви вже маєте API-ключ, його потрібно додати у файл .env 4, 5\.  
* **Вибір SDK:** Рекомендується використовувати **OpenAI Agents SDK** (доступний для Python та TypeScript/Next.js), оскільки він бере на себе складну роботу з WebRTC/WebSockets та керування сесіями 2, 6\.  
* **Формат аудіо:** Модель працює з raw PCM16 аудіо (24кГц, моно) 7, 8\.

### 2\. Реалізація «Перемикача режимів» (Dialogue Mode)

Для вашого сценарію критично важливо керувати тим, коли модель «слухає», а коли ні. Це реалізується через налаштування **Voice Activity Detection (VAD)** та **modalities** 9, 10\.

* **Режим модератора (Тільки виступ):** Встановіть turn\_detection: null. У цьому режимі модель не реагуватиме на звук від користувачів автоматично 11\. Ви вручну ініціюєте виступ моделі командою response.create 11\.  
* **Режим діалогу (Активне слухання):** Перемкніть сесію на turn\_detection: server\_vad. Тепер модель сама розпізнаватиме початок і кінець мовлення гравця та відповідатиме миттєво 9, 12, 13\.

### 3\. Робота з JSON та інструментами (Tools)

Ви повинні створити інструменти, які модель буде викликати автономно 14, 15:

* **get\_question(envelope\_number):** Функція, яка зчитує текст запитання з вашого JSON-файлу за номером конверта 16, 17\.  
* **validate\_answer(user\_answer, question\_id):** Інструмент для порівняння відповіді гравця з правильним варіантом у JSON. Модель може використовувати цей результат, щоб оголосити рахунок 18, 19\.

### 4\. Алгоритм ігрового раунду

#### Крок 1: Оголошення запитання (One-sided)

1. Клієнт (гра) відправляє подію session.update з вимкненим VAD (turn\_detection: null) 11\.  
2. Модель викликає інструмент get\_question, отримує текст і зачитує його.  
3. **Важливо:** На стороні клієнта (веб-додатка) у цей момент мікрофон має бути повністю вимкнений, щоб не витрачати вхідні токени на тишу або шуми 10, 20\.

#### Крок 2: Хвилина на обговорення

1. Модель мовчить, мікрофон клієнта вимкнений.  
2. Таймер на вашому фронтенді відлічує 60 секунд.  
3. Після закінчення часу клієнт ініціює фразу моделі: «Час вичерпано\! Хто відповідатиме?» 11, 21\.

#### Крок 3: Активний діалог та уточнення

1. Клієнт вмикає мікрофон та активує turn\_detection: server\_vad 9, 11\.  
2. Модель слухає ім'я («Відповідає Пан X») і підтверджує: «Добре, Пане X, я вас слухаю».  
3. **Уточнення:** Якщо відповідь неповна, системні інструкції (System Prompt) повинні зобов'язати модель ставити уточнювальні запитання, доки інтент не стане зрозумілим 22, 23\.

#### Крок 4: Оцінка та результат

1. Коли відповідь прийнята, модель викликає validate\_answer.  
2. Модель озвучує правильну відповідь із JSON, додає коментар (small talk) та оновлює рахунок 18\.

### Поради для вашого кейса:

* **Використовуйте GPT-5 Mini:** Це значно зекономить бюджет, оскільки ціна за 1 млн вхідних токенів становить лише $0.25 24, 25\.  
* **Guardrails:** Впровадьте вихідні фільтри (output guardrails), щоб модератор не почав говорити про речі поза грою 26, 27\.  
* **Керування контекстом:** Оскільки гра триває 30 хвилин, використовуйте **контекстну сумаризацію** кожні 3-4 раунди, щоб модель не «забула» правила або рахунок гри через переповнення пам'яті (32k токенів) 28-30.
